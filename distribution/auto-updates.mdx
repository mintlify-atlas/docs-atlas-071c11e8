Electron's built-in `autoUpdater` module lets your app check for, download, and install new releases in the background — without requiring users to manually download and reinstall. It is built on top of the [Squirrel](https://github.com/Squirrel) framework and supports macOS and Windows.

<Note>
`autoUpdater` does not support Linux. On Linux, use the distribution's package manager (`.deb`, `.rpm`, Snap, Flatpak) to distribute updates through the appropriate repository.
</Note>

<Warning>
On macOS, your app **must be code-signed** for `autoUpdater` to work. This is a hard requirement of Squirrel.Mac. See [Code signing](/distribution/code-signing) for setup instructions.
</Warning>

## update.electronjs.org for GitHub releases

The easiest way to ship updates is with [update.electronjs.org](https://github.com/electron/update.electronjs.org) — a free, open-source update service maintained by the Electron team. It works with apps that meet these criteria:

- Runs on macOS or Windows
- Has a public GitHub repository
- Publishes builds to [GitHub Releases](https://docs.github.com/en/repositories/releasing-projects-on-github/managing-releases-in-a-repository)
- Builds are code-signed (required on macOS)

### Setup

<Steps>
  <Step title="Install update-electron-app">
    The `update-electron-app` package is a drop-in Node.js module preconfigured for `update.electronjs.org`. Install it:

    <Tabs>
      <Tab title="npm">
        ```bash
        npm install update-electron-app
        ```
      </Tab>
      <Tab title="yarn">
        ```bash
        yarn add update-electron-app
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Call the updater in your main process">
    Add a single `require` call near the top of your main process file. The module reads your app's `package.json` to determine the GitHub repository and current version automatically.

    ```javascript main.js
    require('update-electron-app')()
    ```

    By default, the module checks for updates at app startup and then every ten minutes. When an update is downloaded, it shows a native dialog prompting the user to restart.
  </Step>

  <Step title="Publish a new release">
    When you publish a new release to GitHub Releases, running apps will detect it on their next check and download it automatically.

    Make sure your release assets include the platform-specific distributables produced by Electron Forge's makers (`.zip` for macOS, `.nupkg` + `RELEASES` for Windows).
  </Step>
</Steps>

## Using a custom update server

If your app is private, hosted outside GitHub, or requires percentage rollouts or authentication, you can run your own update server and point `autoUpdater` at it directly.

Popular self-hosted options:

- **[Hazel](https://github.com/vercel/hazel)** — Simple update server deployable for free on Vercel. Pulls from GitHub Releases and serves via GitHub's CDN.
- **[Nuts](https://github.com/GitbookIO/nuts)** — Also uses GitHub Releases but caches updates on disk and supports private repositories.
- **[electron-release-server](https://github.com/ArekSredzki/electron-release-server)** — Provides a full dashboard for managing releases. Does not require GitHub.
- **[Nucleus](https://github.com/atlassian/nucleus)** — Supports multiple apps and channels with a static file store to minimize hosting costs.

### Complete implementation example

The following example sets up `autoUpdater` with a custom server, checks for updates every minute, and shows the user a restart dialog when a new version is ready.

```javascript main.js
const { app, autoUpdater, dialog } = require('electron')

// Only run the updater in a packaged app, not during development.
if (app.isPackaged) {
  const server = 'https://your-update-server.com'
  const url = `${server}/update/${process.platform}/${app.getVersion()}`

  autoUpdater.setFeedURL({ url })

  // Check for updates every 60 seconds.
  setInterval(() => {
    autoUpdater.checkForUpdates()
  }, 60_000)

  // Notify the user when an update has been downloaded.
  autoUpdater.on('update-downloaded', (event, releaseNotes, releaseName) => {
    const dialogOpts = {
      type: 'info',
      buttons: ['Restart', 'Later'],
      title: 'Application Update',
      message: process.platform === 'win32' ? releaseNotes : releaseName,
      detail: 'A new version has been downloaded. Restart the application to apply the update.',
    }

    dialog.showMessageBox(dialogOpts).then((returnValue) => {
      if (returnValue.response === 0) autoUpdater.quitAndInstall()
    })
  })

  // Log errors.
  autoUpdater.on('error', (message) => {
    console.error('There was a problem updating the application')
    console.error(message)
  })
}
```

<Warning>
Always guard your `autoUpdater` code with `app.isPackaged`. The Squirrel update mechanism is not available in development mode, and calling `autoUpdater.setFeedURL()` in a non-packaged environment will throw an error.
</Warning>

## Key autoUpdater events

| Event | Description |
|---|---|
| `checking-for-update` | Emitted when the updater starts checking for a new version. |
| `update-available` | Emitted when a new version is found. The download starts automatically. |
| `update-not-available` | Emitted when the current version is already up to date. |
| `update-downloaded` | Emitted when the update has finished downloading. Call `autoUpdater.quitAndInstall()` to apply it. |
| `error` | Emitted when an error occurs during the update process. |

## Static storage (serverless)

For apps that do not need a dynamic update server, you can host release metadata files on cloud object storage (for example, S3) and point `autoUpdater` at a static URL.

Install `update-electron-app` and configure it with `UpdateSourceType.StaticStorage`:

```javascript main.js
const { updateElectronApp, UpdateSourceType } = require('update-electron-app')

updateElectronApp({
  updateSource: {
    type: UpdateSourceType.StaticStorage,
    baseUrl: `https://my-bucket.s3.amazonaws.com/my-app-updates/${process.platform}/${process.arch}`,
  },
})
```

Use Electron Forge's S3 Publisher with `macUpdateManifestBaseUrl` (ZIP Maker) and `remoteReleases` (Squirrel.Windows Maker) to publish the required metadata alongside your release artifacts. See the [Forge S3 auto-update guide](https://www.electronforge.io/config/publishers/s3#auto-updating-from-s3) for an end-to-end example.

## Windows: Squirrel.Windows notes

On Windows, Squirrel.Windows launches your app with the `--squirrel-firstrun` argument immediately after installation. During this period Squirrel holds a file lock on your app and `autoUpdater` requests will fail. Avoid checking for updates when `process.argv` includes `--squirrel-firstrun`, or add a short delay:

```javascript main.js
if (!process.argv.includes('--squirrel-firstrun')) {
  // Safe to check for updates here
  autoUpdater.checkForUpdates()
}
```
